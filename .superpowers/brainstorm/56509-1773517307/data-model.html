What Changed From v1

v1 v2 Why
User + Account + VerificationToken Player (name + pin) No auth system needed
Event вҶ’ Game вҶ’ GameParticipant Gone Single game, no multi-tenancy
Work (title, type, year, imageUrl, externalId, wikipediaSlug) Nomination (title, subtitle, imageUrl) Flattened. "Oppenheimer" is just a nomination title, not a separate entity.
Person (name, imageUrl, externalId, wikipediaSlug)
Pick had gameId + userId + categoryId Pick has playerId + categoryId No game abstraction needed

Key Details

  • 4 tables instead of 11
  • Nomination.title вҖ” the work name ("Oppenheimer") or person name ("Cillian Murphy")
  • Nomination.subtitle вҖ” secondary info (e.g., "Cillian Murphy" under Best Picture, or "Oppenheimer" under Best Actor)
  • Player.pin вҖ” hashed short PIN, just enough to prevent casual snooping
  • Category.winnerId вҶ’ points to the winning Nomination
  • Category.isRevealed вҖ” controls leaderboard scoring visibility (same as v1)
  • No game status state machine вҖ” app state is implicit: no winnerId set = picking phase, any winnerId set = live phase
